# 操作日志（5.1）和登录日志（5.2）的区别

**日期**:2026-07-01
**阶段**:5.2 登录日志
**触发问题":你说"都是日志，有什么区别"。

---

## 一句话

> **5.1 记"管理员干了什么"，5.2 记"谁登录了、谁还在线"。**

---

## 详细对比

| 维度 | 5.1 操作日志 | 5.2 登录日志 |
|------|------------|------------|
| **记录对象** | 已登录用户的后台操作（增删改） | 登录/登出事件（含失败） |
| **数据来源** | plugin 自动采集（macro `beforeHandle` + 全局 `onAfterResponse`/`onError`） | `auth/routes.ts` 里手动 `db.insert` |
| **采集点** | `POST/PUT/DELETE` 写接口 | `/login`、`/logout`、`/logout-all` |
| **业务价值** | 审计溯源：谁在什么时候改了什么 | 安全监控：谁在什么时间登录了、失败了、还在线 |
| **表** | `sys_oper_log` | `sys_login_log` |
| **在线状态** | 无 | 有（Redis `online:user:{id}`，带 TTL） |
| **清理策略** | 物理删除 | 物理删除（同事件型表策略） |

---

## 为什么登录日志不也用 plugin 自动采集

登录的时候还没有 JWT token，`ctx.user` 是 undefined。plugin 的 `onAfterResponse` 拿不到用户信息，没法记"谁登录了"。

所以登录日志只能手动 insert——在登录成功/失败的`auth/routes.ts`里拿到 user 后直接写 DB。

---

## 一个场景就搞清楚

```
小王登录系统
  → 5.2 记一条：谁（小王）什么时候（08:00）从哪登录的（IP 192.168.1.1）
  → 在线用户里多一条：小王在线

小王删了一个角色
  → 5.1 记一条：谁（小王）删了什么（role:delete）哪个角色（管理员）
  → 5.2 不记录，跟登录没关系

小王点了登出
  → 5.2 记一条：小王登出了
  → 在线用户删除小王的记录
```

# 开发笔记

> 这里记的是开发过程中的**想法、顿悟、吐槽、反思**，不是规则，不是决策记录。
> 规则去 AGENTS.md，决策去 docs/adr/，踩坑修复去 troubleshooting.md。
> 这里就是**人话写的、给未来的自己看的东西**。

## 这是什么

每条笔记回答一个简单的问题：**这次我搞懂了什么？**

不是教程，不是文档，就是"哦原来如此，我记一下免得忘"。

## 怎么写

- 像发朋友圈或跟朋友解释一样写，不用结构化
- 想到啥写啥，长短随意，一段话也行
- 不要写"本章我们学习"，写"我之前一直以为..."
- 可以是概念理解、踩坑反思、对某个设计的吐槽、某个瞬间的开窍
- 文件名：日期-关键词，比如 `2026-06-14-认证表为什么拆.md`

## 和其他文档的区别

| 文档 | 干啥的 |
|---|---|
| AGENTS.md | 给 AI 看的规矩（"禁止用 any"） |
| docs/adr/ | 重大决策的正式记录（"选 A 不选 B"） |
| docs/troubleshooting.md | 已修复的坑（"报错 X 怎么解"） |
| **docs/notes/**（这里） | **给自己的随笔（"我悟了..."）** |

## 什么时候记

- 突然理解了某个一直搞不懂的概念
- 发现 AI 给的方案/命名不对劲，自己追问出真相
- 对比了几个方案后有了自己的判断
- 踩坑后想通"为什么会踩"
- 任何"这个想法有意思"的瞬间

不用每件事都记，记**让你有"哦"的感觉**的那些就够了。

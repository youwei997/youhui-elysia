# local-fs 路径穿越

`2026-07-05` · 阶段 5 文件存储模块

---

`createLocalFsStorage` 接收一个 `rootDir`，所有文件都存在这个目录下面。

但如果前端传的 key 是 `"../../etc/passwd"` 呢？

```ts
path.resolve("/uploads", "../../etc/passwd")
// → "/etc/passwd"
```

文件就写到根目录外面去了。

一开始没注意到这个问题，直到把 `local-fs` 接进路由，才意识到：**文件上传接口的 key 是用户可控的，不能直接拼路径。**

---

解决思路很简单：写文件前先算一遍绝对路径，确认它在 rootDir 下面。

```ts
const resolveKey = (key: string): string => {
  const fullPath = path.resolve(resolvedRoot, key);
  if (
    !fullPath.startsWith(resolvedRoot + path.sep) &&
    fullPath !== resolvedRoot
  ) {
    throw new Error("Invalid storage key");
  }
  return fullPath;
};
```

`path.resolve` 会吃掉 `..`，所以不管 key 怎么写，`fullPath` 都是真实磁盘路径。然后只要确认它以 `resolvedRoot + "/"` 开头，就说明还在安全区域内。

---

后来查了一下，路径穿越是 OWASP Top 10 里文件上传模块的经典漏洞。这个防护不是业务需求，是**安全底线**。

**总结：任何"用户输入拼文件路径"的场景，第一步就应该做 `path.resolve` + 前缀校验，不要等出事了再补。**
